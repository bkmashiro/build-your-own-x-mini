# 07 — mini-vm

> 一个用纯 Python 实现的最小栈式字节码虚拟机与编译器，支持局部变量帧和反汇编输出。

## 背景

栈式虚拟机是理解解释器最直接的模型之一。它不直接“执行源码”，而是把过程拆成两步：

1. 先把源码解析成语法树。
2. 再把语法树编译成一串字节码。
3. 最后由虚拟机在值栈上逐条执行这些指令。

这种模型并不只是教学玩具。很多真实系统都带有明显的栈机味道：

- JVM 字节码
- CPython 的字节码解释循环
- WebAssembly 的栈式求值语义
- 各种教学语言和脚本语言解释器

这个项目用一个非常小的 Python 子集来展示这条链路，支持：

- 算术表达式
- 变量读写
- `if`
- `while`
- 函数定义与 `return`
- `print(...)`

目标不是复刻 Python，而是把“编译成字节码之后到底怎么跑”的主干过程讲清楚。

## 架构

### 1. 指令集：所有行为都压缩成少量 opcode

虚拟机支持的指令非常少：

- `PUSH`、`POP`
- `ADD`、`SUB`、`MUL`、`DIV`、`MOD`
- `LOAD`、`STORE`
- `JUMP`、`JUMP_IF`
- `CALL`、`RET`
- `PRINT`、`HALT`

这套设计体现了栈机的典型风格：

- 表达式求值时，把中间结果压栈
- 运算指令从栈顶弹出操作数，再把结果压回去
- 控制流通过修改 instruction pointer 跳到别的位置

因此，哪怕没有寄存器、没有复杂对象系统，已经足以表达函数调用、循环和分支。

### 2. 局部变量帧：函数调用的最小运行时模型

每次执行 `CALL` 时，VM 都会创建一个新的 frame。这个 frame 只保存三样东西：

- 当前函数的局部变量字典
- 返回地址
- 函数名

这就是“局部作用域”在运行时的最小表示形式。函数参数在进入 frame 时一次性绑定，之后 `LOAD` / `STORE` 都只操作当前 frame 的局部变量。

虽然这个实现没有闭包、没有对象、没有全局命名空间解析规则，但对理解“为什么函数调用需要栈帧”已经足够。

### 3. 编译器：把结构化语法变成线性字节码

源码的前端没有自己写 parser，而是直接借用 Python 内置 `ast`，然后只接受一个很小的子集节点：

- `Assign`
- `Return`
- `If`
- `While`
- `Expr`
- `Constant` / `Name` / `BinOp` / `Call`

编译过程非常直接：

- 赋值先计算右值，再 `STORE`
- `if` / `while` 先生成条件，再留下待回填的跳转地址
- 函数调用先压入参数，再发出 `CALL`
- `print(x)` 编译成 `PRINT`

因为字节码是线性的，所以分支和循环都需要“回填 jump 目标”。这也是最小编译器里很值得掌握的一个技巧。

## 关键实现分析

### 为什么用栈，而不是寄存器

教学场景里，栈式求值最大的好处是简单。

例如 `a + b * c` 只需要编译成：

1. `LOAD a`
2. `LOAD b`
3. `LOAD c`
4. `MUL`
5. `ADD`

编译器不用考虑寄存器分配，也不用跟踪中间值应该放在哪里。表达式树天然就能后序遍历成一串压栈/出栈动作。

### 条件和循环本质上只是跳转

`if` 和 `while` 看起来是高级语法，但落到 VM 层面都只剩下两种动作：

- 先计算条件
- 再根据结果决定是否跳走

`JUMP_IF` 会从栈顶取出条件值；如果是假值（这里用 `0` 表示），就把 `ip` 改成目标地址。`while` 则是在循环体末尾补一条回跳。

也就是说，结构化控制流只是“带标签的 jump 模板”。

### 函数调用如何落地

`CALL` 执行时做三件事：

1. 从值栈中取出参数
2. 创建新 frame，并把参数名绑定到实参值
3. 保存返回地址，跳到函数起始字节码位置

`RET` 则把返回值压回调用者的栈上，再恢复旧 frame 和 `ip`。这已经完整体现了调用栈最核心的运行时机制。

### 反汇编为什么重要

解释器最难学的地方之一，是源码和运行时之间隔了一层“看不见的动作”。反汇编把这层动作显式打印出来：

- 哪些表达式先压栈
- 分支跳到了哪里
- 调用点和函数入口如何对应

一旦能把源码和字节码一一对上，VM 的执行过程就不再抽象。

## 如何运行

```bash
python3 demo.py
```

`demo.py` 会先打印反汇编结果，再运行三段示例程序：

- 递归 Fibonacci
- 迭代 factorial
- 循环求和

这样可以同时观察“编译后长什么样”和“执行结果是什么”。

## 关键收获

- 栈式虚拟机是理解解释器和字节码系统的优秀起点。
- 表达式求值天然适合转成压栈/出栈序列。
- `if`、`while`、函数调用这些高级结构，落到运行时都可以压缩成很少几条指令。
- 一个几十到一百多行的 VM，就已经能把调用栈、局部变量帧和跳转逻辑讲明白。
