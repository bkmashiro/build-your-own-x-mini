# mini-physics

> 一个极小的 2D 刚体物理核心：AABB 碰撞、冲量解算、固定时间步模拟。

[English](README.md)

---

## 背景

很多游戏物理引擎之所以显得复杂，是因为它们同时处理了 broad phase、接触流形、休眠、摩擦、关节和多种形状。真正的核心循环其实很短：

- 根据速度和重力积分位置
- 检测重叠
- 计算碰撞法线和穿透深度
- 将物体分离，并沿法线施加冲量

这个项目只保留这条最小路径，并使用轴对齐包围盒（AABB），让碰撞数学保持直观。

---

## 结构

```text
step(world, dt)
  -> 积分重力和速度
  -> 检测重叠的 AABB 对
  -> 逐个解算碰撞
       - 计算法线方向上的相对速度
       - 根据恢复系数施加冲量
       - 做位置修正
```

`Body` 保存 `position`、`velocity`、`size` 和逆质量。静态物体只需要设置 `invMass = 0`，因此同一套解算器就能覆盖动态-静态和动态-动态碰撞。

碰撞检测会分别计算 x/y 两个轴的重叠量：

- 任一轴重叠量小于等于 0，说明没有碰撞
- 否则选择重叠更小的轴作为接触法线
- 只有当两个物体正朝彼此运动时才施加冲量

这就形成了一个很小但足够真实的盒体碰撞解算器。

## 运行方式

```bash
npm test
npm run demo
```

这里直接使用 Node 内置测试器配合 `--experimental-strip-types`，不需要额外的 TypeScript 构建步骤。

---

## 省略内容

- 旋转和角速度
- 摩擦与静止接触稳定
- 连续碰撞检测
- broad-phase 加速结构

这些都是生产级物理引擎的重要部分，但理解刚体积分和基于冲量的碰撞解算并不需要它们。
