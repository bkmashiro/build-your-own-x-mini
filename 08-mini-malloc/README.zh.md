# 08 — mini-malloc

> 一个用纯 Python 实现的最小堆内存分配器，使用空闲链表、first-fit 分配和相邻块合并。

## 背景

动态内存分配的本质，是把一整块连续内存拆成很多可变大小的小块，再在需要时分出去、释放后收回来。工业级 allocator 往往非常复杂，但它们的主干思想其实可以压缩成几个核心问题：

- 堆是一段连续地址空间
- 每个 block 前面都要有 header，记录元数据
- 空闲块需要被某种结构追踪起来
- 分配时要找到合适的块，必要时拆分
- 释放时要尽量把碎片重新合并

这个项目用 Python `bytearray` 模拟一段 heap，把这些机制完整展示出来。

## 架构

### 1. Block Header：每块内存前面都带一段元数据

每个 block 的头部固定 12 字节，包含三项：

- `size`
- `free_flag`
- `next_ptr`

也就是题目里要求的 `[size | free_flag | next_ptr]`。payload 紧跟在 header 后面，因此返回给用户的“指针”其实就是 `header_offset + header_size`。

这里的 `next_ptr` 让所有 block 形成一个链表，而且链表顺序与 heap 中的物理顺序一致。这一点很重要，因为后面的 coalescing 要依赖“相邻块是否真正连续”。

### 2. Free List：用 first-fit 找可用块

分配器维护的不是“所有空闲大小的复杂索引”，而是最简单的一条链表。`malloc(size)` 从头扫描，找到第一个满足条件的 free block 就使用它。

这就是 **first-fit**：

1. 从链表头开始找
2. 遇到第一个足够大的空闲块就停下
3. 如果这个块太大，就拆分

它不是最先进的策略，但非常适合教学，因为行为容易观察：

- 早期空闲块会优先被复用
- 会出现外部碎片
- 需要 coalescing 来恢复大块空间

### 3. Heap 顺序与链表顺序一致

这个实现没有把 block 从链表里摘出来再单独挂 free list，而是让所有 block 始终按地址顺序串起来。这样有两个好处：

- 结构简单，代码更短
- 合并时只要检查“当前块末尾是否正好等于下一个块起点”

如果两者连续且都空闲，就能直接拼成一个更大的块。

## 关键实现分析

### `malloc(size)`：分配与拆分

当 `malloc` 找到一个足够大的 free block 后，会分两种情况：

- **刚好够用或剩余太小**：整块拿走
- **明显更大**：把尾部拆成一个新的 free block

拆分的前提是剩余空间还能容纳一个完整 header 再加至少 1 字节 payload，否则拆了也没有意义。

这一步是 allocator 的核心，因为“如何切块”直接决定了后续碎片情况。

### `free(ptr)`：标记释放，再尝试合并

`free` 本身并不复杂：

1. 根据用户指针回到 block header
2. 把 `free_flag` 设为 1
3. 扫描链表，尝试合并相邻空闲块

真正重要的是第三步。如果只标记 free 不合并，heap 很快就会被很多小碎片填满，即使总空闲空间够，也可能找不到一块足够大的连续区域。

### Coalescing：为什么能修复碎片

假设有两个 block：

- 当前块大小为 `size`
- 下一个块起点正好是 `current + header + size`

这说明它们在物理上紧挨着。如果两个块都空闲，就可以把它们合成一个更大的 free block，并跳过原来的下一个节点。

这就是 allocator 里最基本的碎片修复机制。很多更复杂的分配器，本质上也离不开这个思想。

### `realloc(ptr, size)`：优先原地扩容

`realloc` 的理想情况是“不搬家”：

- 如果原 block 已经足够大，直接复用，必要时缩小并拆分
- 如果后面紧挨着的是 free block，且合并后足够大，就原地扩容

只有这两条都失败时，才退化为：

1. `malloc(new_size)`
2. 复制旧数据
3. `free(old_ptr)`

这也很贴近真实 allocator 的思路：尽量避免搬动数据。

### 可视化输出为什么有帮助

堆分配器最抽象的地方，是“看不到内存怎么被切”。`layout()` 把每个块打印成：

- 起始偏移
- `free` / `used`
- payload 大小

这样分配、释放、复用、合并的效果就能直接观察出来，非常适合教学 demo。

## 如何运行

```bash
python3 demo.py
```

`demo.py` 会演示：

- 连续分配多个 block
- 释放中间块后，first-fit 如何优先复用它
- 释放相邻块后如何自动 coalesce
- `realloc` 如何在保留 payload 的前提下扩容

## 关键收获

- 一个最小 allocator 只需要 header、链表和拆分/合并规则，就能把核心机制讲清楚。
- first-fit 很容易实现，也很容易观察到碎片行为。
- coalescing 是释放后恢复大块连续空间的关键。
- `realloc` 的本质是“能原地扩就原地扩，实在不行再搬迁”。
