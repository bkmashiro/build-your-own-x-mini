# 06 — mini-regex

> 一个用纯 Python 实现的最小 Thompson-NFA 正则引擎 —— 核心代码控制在 200 行以内。

## 背景

很多人第一次接触正则表达式时，会把它理解成“某种神秘的回溯黑盒”。但对教学来说，更清晰的方式其实是把正则看成一个图：

1. 先把模式串解析成运算符结构。
2. 用 **Thompson 构造法** 把它转换成 **NFA（非确定有限自动机）**。
3. 运行时维护“当前所有可能的状态”，并通过 **epsilon-closure（epsilon 闭包）** 扩展零宽跳转。

这个项目就是把这条路线完整落地。它支持：

- `.` 任意字符
- `*`、`+`、`?`
- `|`
- `()`
- `[abc]`
- `^` / `$`
- 简单捕获组

目标不是兼容完整的 Python `re`，而是把“正则到底怎么工作”的主干机制讲清楚。

## 架构

整个实现只有三层。

### 1. 解析器：把字符串变成语法树

核心文件先用一个很小的递归下降解析器，把模式串转成树状结构。主要节点包括：

- **alternation**：`a|b`
- **concatenation**：`ab`
- **repetition**：`a*`、`a+`、`a?`
- **group**：`(ab)`
- **atom**：普通字符、`.`、字符类、锚点

为什么先做语法树？因为正则运算符本质上有优先级：

- `* + ?` 只作用在前一个 atom 上
- 拼接的优先级高于 `|`
- 括号可以显式改变结合方式

先把结构整理清楚，后面的 NFA 构造就会非常直接。

### 2. NFA 构造：把每个节点变成一段小图

Thompson 构造法的关键思想是：每种正则运算符都可以对应成一个固定模板的图片段，每个片段都只有：

- 一个入口状态
- 一个出口状态

然后通过 epsilon 边把这些片段拼起来。

例如：

- **字面量 `a`**：一条消费字符的边
- **拼接 `ab`**：把 `a` 的出口用 epsilon 连到 `b` 的入口
- **选择 `a|b`**：新建一个分叉入口和汇合出口
- **`a*`**：入口既可以直接跳过，也可以进入 `a`，而 `a` 的出口既可以回环也可以退出
- **`a+`**：和 `*` 类似，但第一次不能跳过
- **`a?`**：要么走 `a`，要么直接跳过
- **分组 `(ab)`**：在进入和离开子图时挂上“捕获开始/结束”的标记

这就是 Thompson 算法最美的地方：规则统一、组合自然，而且特别适合教学。

### 3. NFA 执行：不是“选一条路”，而是“同时保留所有可能”

运行时，正则引擎不会从多条路径里“猜一个”，而是维护一个配置集合：

- 当前可能处于哪些状态
- 每条路径目前记录了哪些分组边界

每读一个字符，都分成两步：

1. 先做 **epsilon-closure**
   把所有不消耗字符的边（包括普通 epsilon、`^`、`$`、分组标记边）尽可能走完。
2. 再根据当前字符，沿着可消费字符的边推进到下一批状态。

这正是 NFA 执行的核心。所谓“非确定性”，并不是随机选，而是把所有分支都保留下来一起推进。

## 关键实现分析

### Thompson 构造法为什么适合这个项目

如果直接写递归回溯匹配器，代码通常会把“解析逻辑”和“执行逻辑”混在一起，初学者很难分清楚：

- 哪一部分是在理解正则结构
- 哪一部分是在处理输入文本

而 Thompson 路线把两件事明确拆开：

- **编译阶段**：正则 -> NFA 图
- **执行阶段**：文本 -> 活跃状态集合

这样代码结构会非常干净，也更容易解释每个运算符到底做了什么。

### epsilon-closure 是整个运行时的灵魂

闭包步骤的含义是：

> 在当前位置不消耗任何字符的前提下，我最多还能走到哪些状态？

为什么它这么重要？

- `a*` 之所以能匹配零次，是因为可以通过 epsilon 直接跳过
- `a?` 之所以可选，也是因为有 epsilon 分支
- `a|b` 之所以能分叉，是因为起点先通过 epsilon 同时连向两个分支
- 捕获组之所以能记录边界，是因为“开始/结束”本身就是零宽动作
- `^` 和 `$` 之所以是锚点，是因为它们不消耗字符，只检查当前位置是否合法

所以从教学角度看，理解 epsilon-closure，基本就理解了 NFA 正则执行的一半。

### 分组是怎么记录的

这个实现没有引入复杂的回溯栈，而是把捕获信息直接挂在状态配置上：

- 进入某个 group 时，记录开始位置
- 离开某个 group 时，记录结束位置

由于不同分支可能拥有不同的捕获结果，闭包和状态推进时都把“状态 + 当前 spans”一起携带。

这说明一个很重要的事实：

> 捕获组并不是 NFA 模型之外的特殊机制，它完全可以作为状态流的一部分。

### 锚点如何融入同一个模型

`^` 和 `$` 在这个实现里也是零宽边：

- `^` 只有在 `pos == 0` 时允许通过
- `$` 只有在 `pos == len(text)` 时允许通过

也就是说，锚点不是额外分支逻辑，而只是“带条件的 epsilon 转移”。这让整体模型保持统一。

## 如何运行

```bash
python3 demo.py
```

`demo.py` 会先展示几组内置例子，然后进入一个简单交互模式。你可以输入：

- pattern
- text

程序会分别输出：

- `search()` 的结果
- `fullmatch()` 的结果
- 捕获组内容和 span

这样很适合边改模式边观察状态机行为产生的效果。

## 关键收获

- 正则的核心可以压缩成一个很小的状态机实现，不需要一开始就接触庞大的工业级引擎。
- Thompson 构造法最大的优点是“规则统一”，每个运算符都只是一个可组合的小图模板。
- 真正驱动匹配的是 epsilon-closure：先扩展零宽路径，再消费输入字符。
- 捕获组、锚点、可选和重复，其实都能自然落在同一个 NFA 执行框架里。
- 这个版本当然省略了大量工业特性，比如转义语义扩展、范围字符类、懒惰量词、回溯优先级细节等，但教学主线已经完整。
